# V5.3 PIC-2B Pricing Implementation Report

## Scope

Implemented Product Detail Pricing Tab V1 only.

This phase does not introduce a full pricing engine, landed cost logic, freight pricing, AI pricing intelligence, or duplicate product tables. Product Library remains the single source of truth.

## Changed Files

- `public/app.js`
  - Added Product Detail `Pricing` tab.
  - Added Pricing Summary, Variant Price Grid, and Pricing Rule Summary UI.
  - Added permission-aware pricing display.
  - Improved Product Intelligence List pricing status display to uppercase business statuses.
- `public/styles.css`
  - Added layout styles for pricing summary, variant pricing grid, and pricing rule summary.
- `src/server.mjs`
  - Extended Product Intelligence pricing summary response using existing product variant pricing fields.
  - Preserved supplier currency separately from selling currency.
  - Added `converted_cost_currency` to sensitive pricing fields so Sales users do not receive internal cost conversion data.
- `tests/integration.test.mjs`
  - Added regression coverage for CNY supplier cost, USD converted cost, selling price visibility, Sales cost redaction, Quote selling-price usage, and PI export cost protection.

## Database Changes

None.

PIC-2B reuses existing fields:

- `supplier_cost`
- `supplier_currency`
- `converted_cost`
- `reference_price`
- `pricing_status`
- `pricing_rule_id`

The UI/API maps these into the requested display concepts:

- Supplier Cost: `supplier_cost`
- Supplier Currency: `supplier_currency`
- Converted Cost: `converted_cost`
- Converted Cost Currency: fixed display as `USD`
- Recommended Selling Price: `reference_price`
- Selling Currency: fixed display as `USD`
- Pricing Status: `pricing_status`

## API Changes

No new API endpoint was added.

Enhanced existing Product Intelligence detail response:

- `GET /api/product-intelligence/products/:id`

The response now includes a richer `pricing_summary` structure for the Pricing tab:

- Reference selling price range
- Selling currency
- Pricing status
- Variant pricing rows
- Permission-aware supplier cost and converted cost fields

Sales-safe redaction remains enforced server-side.

## Permission Behavior

Owner / Admin can see:

- Supplier Cost
- Supplier Currency
- Converted Cost
- Pricing Rule
- Recommended Selling Price

Sales can see:

- Recommended Selling Price
- Selling Currency
- Pricing Status

Sales cannot receive:

- Supplier Cost
- Supplier Currency
- Converted Cost
- Converted Cost Currency
- Pricing Rule ID
- Cost snapshots

## Currency Flow Fix

Supplier currency is preserved and no longer treated as the selling currency.

Example validated by test:

- Supplier Cost: `CNY 156`
- Exchange Rate: `7.2`
- Converted Cost: `USD 21.67`
- Recommended Selling Price: `USD 43.00`

## Quote / PI Protection

Verified:

- Quote Builder uses recommended selling price / reference price.
- Sales quote response does not expose cost snapshot fields.
- PI Excel export does not include supplier cost, supplier currency, or internal cost fields.

## Known Limitations

- Selling currency is currently displayed as USD because the existing data model does not yet include a separate per-product selling currency field.
- Converted cost currency is displayed as USD based on the current price calculation flow.
- Pricing rule editing and advanced pricing engine controls are intentionally not included in PIC-2B.
- Landed cost, freight pricing, AI pricing intelligence, and quote currency conversion are future phases.
