# V5.3 Opportunity Intelligence V2 — AI Customer Discovery Assistant MVP Report

## Scope Completed

Implemented the AI-assisted customer discovery planning layer inside the existing Opportunity Intelligence workflow.

This release does not execute external discovery. It helps users turn a natural-language target customer request into a structured, reviewable Customer Discovery Plan before any future search execution.

## Database Changes

Additive only. No existing customer table was duplicated.

New tables:

- `customer_type_profiles`
  - Stores dynamic customer categories, e.g. Restaurant Furniture Distributor, Restaurant Interior Design Company, Restaurant Owner.

- `customer_type_score_dimensions`
  - Stores customer-type-specific scoring dimensions and weights.
  - Example: Distributor uses Business Match / Purchase Potential / Company Size / Contact Availability.
  - Example: Design Company uses Project Capability / Restaurant Experience / Client Base / Contact Availability.

- `customer_discovery_requests`
  - Stores the original user request, generated search plan, guidance, scoring profile snapshot, AI execution log reference, and cost log reference.

No duplicate customer database was created. Discovery output is a planning record that flows toward the existing Customer List and AI Processing workflow.

## API Changes

New APIs:

- `GET /api/customer-discovery/config`
  - Returns dynamic customer type profiles and scoring dimensions.

- `GET /api/customer-discovery/requests`
  - Returns recent discovery planning requests.

- `POST /api/customer-discovery/analyze`
  - Analyzes natural-language user input and creates a Customer Discovery Plan.

- `POST /api/customer-discovery/generate-plan`
  - Generates a structured search plan using the same governed planning layer.

All APIs reuse existing Opportunity Intelligence permission checks.

## AI / Cost Control Integration

- No external AI is called on page load.
- Analysis runs only when the user clicks:
  - Analyze Requirement
  - Generate Search Plan
- AI Business Brain is used through `runAiAction(...)`.
- Provider is `rules`.
- AI Execution Logs are created.
- AI Cost Control logs are created with zero-cost rules execution.
- AI output remains advisory and does not modify Customer records automatically.

## Behavior Implemented

Natural-language examples supported:

- `Find restaurant furniture suppliers in California`
  - Target Customer Type: Restaurant Furniture Distributor
  - Country: USA
  - Region: California
  - Keywords: restaurant furniture supplier, hospitality furniture distributor, commercial seating supplier

- `Find restaurant design companies in Texas`
  - Target Customer Type: Restaurant Interior Design Company
  - Dynamic scoring profile: Project Capability, Restaurant Experience, Client Base, Contact Availability

- `Find restaurant customers`
  - Marked as broad/incomplete.
  - Guidance asks for target country, company size, product category, and decision maker requirement.

## Remaining Limitations

- No Google Maps API integration.
- No Apollo integration.
- No LinkedIn integration.
- No website crawler.
- No autonomous search execution.
- No automatic customer creation from discovered results.
- Rules-based analysis is intentionally simple and safe for MVP.
