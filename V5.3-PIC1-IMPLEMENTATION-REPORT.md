## 3. API Changes

Added a read-only Product Intelligence API boundary under:

```text
/api/product-intelligence/*
```

### New APIs

#### `GET /api/product-intelligence/products`

Purpose:

Return Product Intelligence list data from existing Product Library records.

Returns:

- Image
- Product name
- SKU
- Category
- Status
- Visibility
- AI confidence if available
- Data quality / readiness if available
- Capability flags
- AI integration metadata showing no external provider call

Supplier confidential fields are redacted for unauthorized roles.

#### `GET /api/product-intelligence/products/:id`

Purpose:

Return structured Product Intelligence detail.

Returns:

- Basic information
- Attributes
- Variants
- Media
- Pricing summary
- AI profile
- Source information, permission controlled
- Quality summary
- Product relationships

Supplier cost and source supplier data are visible only to authorized roles.

#### `GET /api/product-intelligence/products/:id/quality`

Purpose:

Return product readiness and AI readiness information.

Returns:

- Missing fields
- Data quality score
- Proposal readiness status
- Readiness checks
- AI readiness summary

#### `GET /api/product-intelligence/context/products/:id`

Purpose:

Prepare clean product context for future AI modules.

Intended future consumers:

- AI recommendation
- Quote Builder
- PI Builder
- Sales Assistant

Rules:

- No external AI provider call.
- No source-of-truth mutation.
- Supplier confidential data redacted for unauthorized roles.

## 8. Risks

### A. Existing Monolithic Server

`src/server.mjs` remains large. PIC-1 added helper functions and routes inside the existing file to avoid a larger refactor.

Future risk:

Product Intelligence logic may become harder to maintain if more phases continue to grow in the same file.

Recommended future action:

Move Product Intelligence read models into a dedicated service file during a planned refactor phase.

### B. Quality Score Reuse

PIC-1 reuses current product readiness logic as data quality.

Future risk:

Product Intelligence may eventually need separate scores for:

- Quote readiness
- PI readiness
- AI matching readiness
- Website readiness

Recommended future action:

Add dedicated score records only when the business rules diverge from current readiness.

### C. Pricing Summary Scope

PIC-1 exposes pricing summaries from existing variant/reference price fields.

Future risk:

Users may confuse Product Intelligence pricing summary with the pricing engine.

Control:

Product Price Engine remains the calculation owner. Product Intelligence is read-only in PIC-1.

### D. Source Information Sensitivity

PIC-1 returns source information only to authorized users.

Future risk:

Future AI context expansion could accidentally include supplier source/cost data in customer-facing prompts.

Control:

All future AI context builders must continue to use server-side role redaction and prompt-specific context filtering.
