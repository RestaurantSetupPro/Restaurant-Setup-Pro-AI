## 1. Business Pricing Workflow

Restaurant Setup Pro sells hospitality furniture globally.

Products are sourced from suppliers and factories, then quoted to customers in different countries and currencies.

The pricing model must support this full flow:

```text
Supplier Cost
  ↓
Currency Conversion
  ↓
Cost Calculation
  ↓
Margin Rules
  ↓
Recommended Selling Price
  ↓
Customer Quote Price
  ↓
PI Price
```

### 1.1 Pricing Ownership

The Product Price Engine owns deterministic price calculation.

Quote Builder owns final quote line calculation.

PI Builder owns customer-facing PI output.

AI may explain, warn, and recommend. AI must not silently set final prices.

### 1.2 Core Rule

Supplier cost is internal.

Customer quote price is external.

They must never be mixed in the UI, API response, Quote, PI, export, email, WhatsApp message, or AI customer-facing output.

## 2. Data Model Recommendation

The future implementation should reuse the current Product Library, Product Variants, Product Price Engine, Quote Builder, and PI Builder.

No duplicate product or price system should be created.

### 2.1 Product Master

Product-level pricing should be treated as a summary only.

Recommended product-level fields:

| Field | Purpose |
| --- | --- |
| `product_id` | Product master reference |
| `default_currency` | Preferred selling currency for product display |
| `reference_price_min` | Lowest active variant reference price |

| `price_display_mode` | Starting From / Price Range / Request Quote |

| `pricing_updated_at` | Last pricing update timestamp |
| `pricing_updated_by` | User who last updated or recalculated pricing |

Product-level price should not override variant-level pricing unless the product has no variants.

### 3.2 Supplier Cost Status

| Status | Meaning |
| --- | --- |
| Current | Confirmed current supplier cost |
| Estimated | Cost is estimated or imported from old sheet |
| Expired | Cost is older than configured validity period |
| Needs Confirmation | Cost is missing or unreliable |

### 3.3 Confidentiality Rules

Supplier cost is confidential.

Sales and VA must not receive supplier cost from API responses.

Sales and VA must not see:

- Supplier cost amount
- Supplier currency, if it reveals purchasing data
- Supplier code
- Supplier quote reference
- Purchase cost history
- Profit margin
- Converted cost
- Internal margin calculation

Product Manager / Admin / Owner can view these according to role permissions.

### 3.4 Supplier Cost Source

Supplier cost may come from:

- Supplier Excel import
- Manual product/variant entry
- Supplier quotation
- Supplier price list
- Future supplier portal

Each cost should keep its source reference where possible:

- Source file
- Source sheet
- Source row
- Import batch
- Imported by
- Imported date

## 4. Currency Architecture

### 4.1 Supported Currencies

Initial supported currencies:

- CNY
- USD
- EUR
- MYR

Future supported currencies may include:

- GBP
- CAD
- AUD
- SGD
- AED
- SAR

### 4.2 Base Currency

Recommended internal base currency:

```text
USD
```

Reason:

- International hospitality furniture sales commonly quote in USD.
- PI and export documents often use USD.
- Many international customers expect USD reference pricing.

The system may still support product costs in CNY and customer quotes in USD/EUR/MYR.

### 4.3 Exchange Rate Direction

The system must store exchange rate direction clearly.

Recommended standard:

```text
1 USD = X Supplier Currency
```

Example:

```text
Supplier Cost: CNY 150
Exchange Rate: 1 USD = 6.8 CNY
Converted Cost: USD 22.06
```

Formula:

```text
Converted Cost USD = Supplier Cost CNY / 6.8
```

### 4.4 Exchange Rate Fields

Recommended fields:

| Field | Purpose |
| --- | --- |
| `from_currency` | Supplier/original currency |
| `to_currency` | Base or quote currency |
| `exchange_rate` | Numeric rate |
| `rate_direction` | Example: `USD_TO_CNY` |
| `exchange_rate_source` | Manual / Bank / API / Supplier / Imported File |
| `exchange_rate_date` | Date rate was set |
| `manual_override` | Whether manually set |
| `override_reason` | Reason for manual rate |
| `created_by` | User who set rate |
| `created_at` | Timestamp |

### 4.5 Conversion History

Future entity:

```text
currency_conversion_history
```

Purpose:

- Track exchange rates used in historical price calculations.
- Prevent old quotes from changing when exchange rates change.
- Support audit of margin changes over time.

### 4.6 Currency Rule

Quote and PI must store the currency used at the time of generation.

Historical Quote/PI totals must not recalculate automatically because exchange rates changed later.

## 5. Cost Calculation Layer

Cost calculation starts from supplier cost and may later include more components.

### 5.1 V1 Cost Calculation

V1 formula:

```text
Converted Cost = Supplier Cost / Exchange Rate
```

Example:

```text
CNY 150 / 6.8 = USD 22.06
```

### 5.2 Future Landed Cost Calculation

Future formula:

```text
Landed Cost =
  Converted Supplier Cost
  + Estimated Packaging Cost
  + Domestic Freight
  + Export Handling
  + International Freight Allocation
  + Duty / Tax Allocation
  + Payment / Bank Fees
```

Do not implement full landed cost in the next phase unless separately approved.

For V5.3 Lite, keep pricing engine focused on:

- Supplier cost
- Currency conversion
- Margin rule
- Recommended selling price
- Quote price snapshot

## 6. Pricing Rule Engine

The Pricing Rule Engine determines recommended selling price.

It must remain deterministic.

AI may explain pricing risk but must not calculate hidden final price outside the engine.

### 6.1 Rule Inputs

Recommended rule matching inputs:

- Product category
- Product subcategory
- Supplier
- Customer type
- Customer country
- Quote currency
- Sales channel
- Product status
- Variant status
- Cost confidence

### 6.2 Margin Methods

#### A. Cost Multiplier

Formula:

```text
Recommended Selling Price = Converted Cost × Multiplier
```

Examples:

| Category | Multiplier |
| --- | --- |
| Restaurant Chair | 3.2 |
| Booth Seating | 2.8 |
| Custom Furniture | 2.5 |

Example:

```text
Converted Cost: USD 22.06
Multiplier: 3.2
Recommended Selling Price: USD 70.59
Rounded Selling Price: USD 71.00
```

#### B. Gross Margin

Formula:

```text
Gross Margin = (Selling Price - Cost) / Selling Price
Selling Price = Cost / (1 - Gross Margin)
```

Example:

```text
Cost: USD 100
Target Gross Margin: 45%
Selling Price = 100 / (1 - 0.45) = USD 181.82
```

Use case:

- More finance-oriented margin control.
- Better when leadership thinks in margin percentage.

#### C. Category-Based Rules

Example:

| Product Category | Rule |
| --- | --- |
| Dining Chair | Cost × 3.2 |
| Table Base | Cost × 2.8 |
| Table Top | Cost × 2.6 |
| Booth Seating | Cost × 2.8 |
| Custom Furniture | Cost × 2.5 |
| Outdoor Furniture | Cost × 2.7 |

#### D. Customer-Type Rules

Example:

| Customer Type | Margin Behavior |
| --- | --- |
| Restaurant Owner | Higher margin |
| Furniture Distributor | Lower margin |
| Designer | Medium margin / trade discount |
| Contractor | Project-based margin |

#### E. Country Rules

Example:

| Country / Region | Pricing Behavior |
| --- | --- |
| USA | USD quote, DDP possible, margin includes higher service burden |
| Malaysia | MYR or USD quote, regional competition considered |
| Europe | EUR quote, compliance/logistics buffer may apply |
| Middle East | USD quote, project margin may be higher |

### 6.3 Rule Priority

Recommended priority order:

1. Manual price override
2. Customer-specific contract price
3. Customer type + category rule
4. Country + category rule
5. Supplier + category rule
6. Category rule
7. Global default rule
8. Missing rule → Needs Pricing Review

### 6.4 Rounding Rules

Supported rounding examples:

- Round to nearest `1`
- Round to nearest `5`
- Round to nearest `10`
- Round up to `.99`
- Round to custom increment

Hospitality furniture recommendation:

- B2B product list price: round to nearest USD 1 or USD 5.
- Large custom furniture: round to nearest USD 10.
- PI line item price: use exact quote price chosen by Sales.

### 6.5 Manual Override

Manual override must be tracked.

Required fields:

- Override price
- Override reason
- Override by
- Override date
- Previous calculated price

Only authorized roles should override product reference price:

- Owner
- Admin
- Product Manager
- Purchasing / General Manager, future

Sales can adjust quote unit price if allowed by business process, but cannot change Product Library reference price.

## 8. Country Pricing

Country impacts price through:

- Quote currency
- Shipping cost
- Tax / duty assumption
- Service burden
- Competitive market margin
- DDP/CIF/FOB behavior

### 8.1 Country Default Currency

Examples:

| Country / Region | Default Quote Currency |
| --- | --- |
| USA | USD |
| Malaysia | MYR or USD |
| Europe | EUR |
| Middle East | USD |
| Canada | USD or CAD |
| Australia | USD or AUD |

### 8.2 Shipping and Trade Term

Country pricing should not silently include freight unless the trade term requires it.

Rules:

- EXW/FOB: product price normally excludes international freight.
- CIF: price may include freight to destination port if freight is confirmed.
- DDP: quote may include door delivery, duty/tax assumptions, and higher risk buffer.

If freight is not confirmed:

```text
Freight To Be Quoted
```

Do not fake freight.

### 8.3 Tax and Duty

For V5.3 Lite:

- Do not calculate customs duty automatically unless future rules are approved.
- PI should state trade term clearly.
- Any tax/duty inclusion must be explicit.

Future:

- Country-level duty/tax estimate table.
- Trade-term-specific landed price estimator.

## 9. Quote / PI Integration

### 9.1 Quote Uses Customer Selling Price

Quote Builder must use:

- Recommended selling price
- Customer-specific selling price
- Manually edited quote price

Quote Builder must not expose:

- Supplier cost
- Converted cost
- Margin
- Supplier notes
- Purchase cost history

### 9.2 PI Uses Final Quote Price

PI must record:

- Product price
- Currency
- Quantity
- Total
- Discount, if applicable
- Freight
- Other charges
- Grand total
- Deposit
- Balance

PI must not contain:

- Supplier cost
- Margin
- Internal cost calculation
- Supplier code
- Supplier notes

### 9.3 Quote/PI Snapshot Rule

When Quote/PI is generated:

The system must snapshot customer-facing commercial values.

Reason:

- Product prices may change later.
- Exchange rates may change later.
- Supplier cost may change later.
- Historical quote/PI must remain accurate.

### 9.4 Price Recalculation Rule

Product price recalculation should not automatically change existing quotes or PIs.

If user wants to update an existing quote:

- Show current quote price.
- Show new recommended price.
- Require manual confirmation.
- Create new quote version.

### 9.5 Custom Quote Items

Custom Quote Items may not have supplier cost or product reference price.

Rules:

- Sales manually enters customer quote price.
- Optional internal estimated cost may be added later for margin review.
- Custom quote item price is stored in quote/order snapshot.
- Custom item does not create Product Library record unless user separately chooses Save as Product Draft.

## 11. AI Usage Rules

### 11.1 AI Can Use

AI may use:

- Product information
- Product category
- Variant options
- Sales-safe price range
- Customer type
- Customer country
- Quote context
- Product readiness
- Cost range only for authorized internal users
- Margin rule summaries only for authorized internal users

### 11.2 AI Can Assist With

AI may:

- Explain why a product needs pricing review.
- Recommend whether price should be reviewed before quote.
- Detect suspicious cost/price gaps.
- Summarize product value for sales.
- Suggest quote remarks.
- Suggest PI wording.
- Recommend product alternatives based on budget.
- Warn that quote price is below recommended margin.

### 11.3 AI Cannot

AI must not:

- Expose supplier cost to Sales, VA, or customer.
- Write final price silently.
- Change Product Library reference price without approval.
- Change quote unit price without user confirmation.
- Override Product Price Engine.
- Generate customer-facing explanation mentioning internal margin.
- Call paid provider without AI Cost Control.

### 11.4 AI Pricing Intelligence Future

Future AI Pricing Intelligence should be advisory.

Example outputs:

- “This product has no confirmed supplier cost.”
- “This price was manually overridden 42 days ago.”
- “Customer is a distributor; recommended margin may be lower.”
- “DDP USA quote may need freight/risk buffer.”
- “Quote line is below minimum margin policy; manager approval recommended.”

AI output should be stored as:

- Recommendation
- Analysis
- Draft
- Review record
- Log

Not as source-of-truth price.

## 15. Risks

### 15.1 Supplier Cost Leakage

Risk:

Supplier cost may appear in Sales UI, AI context, export, Quote, PI, email, or WhatsApp message.

Mitigation:

Use server-side permission redaction. Test Sales/VA responses and customer-facing outputs.

### 15.2 Quote/PI Regression

Risk:

Pricing changes may break existing quote totals or PI exports.

Mitigation:

Quote Builder and PI Builder must remain protected core modules. Run regression tests after every pricing change.

### 15.3 Exchange Rate Confusion

Risk:

Users may enter exchange rate in the wrong direction.

Mitigation:

UI must clearly display:

```text
1 USD = X CNY
```

And show calculated preview immediately.

### 15.4 Margin Method Confusion

Risk:

Multiplier and gross margin are often confused.

Mitigation:

Pricing rule UI must label formula clearly and show example calculations.

### 15.5 Historical Quote Mutation

Risk:

Changing product price could unintentionally affect old quotes.

Mitigation:

Always snapshot quote/PI price values. Never recalculate historical documents unless user creates a new version.

### 15.6 Overbuilding Pricing Too Early

Risk:

Building landed cost, country tax, contract pricing, and AI pricing at once could make the system too complex.

Mitigation:

Start with supplier cost, conversion, margin rule, recommended selling price, and quote snapshot.

## 17. Final Principle

Pricing is a business-control system, not an AI playground.

The system should calculate prices deterministically, protect supplier cost rigorously, snapshot customer-facing prices safely, and allow AI only to assist with explanations, warnings, and reviewed recommendations.
