## 3. Database Changes

Additive migration only. No existing business tables were rebuilt or removed.

New tables:

- `ai_prompt_templates`
  - Stores prompt key, version, module, action, variables, output format, template text, and active status.

- `ai_context_snapshots`
  - Stores structured business context snapshots used by AI actions.
  - Includes source references and context hash.

- `ai_execution_logs`
  - Stores unified AI execution records.
  - Includes module, action, user, provider, model, prompt version, context snapshot, status, cost fields, and errors.

Seeded prompt:

- `v53.foundation.mock.v1`

## 4. API Changes

New internal/admin APIs:

- `POST /api/ai-brain/actions/run`
  - Runs a shared AI foundation action.
  - Requires Admin or Owner.
  - Uses mock/rules provider only in Phase 1.
  - Passes through AI Cost Control.
  - Saves context snapshot and execution log.

- `GET /api/ai-brain/status`
  - Returns AI Brain provider, prompt, context, and execution log status.
  - Requires Debug Center permission.

Updated API:

- `GET /api/debug/system`
  - Now includes `aiBusinessBrain` status.

## 7. Known Limitations

- Phase 1 intentionally uses only `mock` and `rules` providers.
- OpenAI, Gemini, Claude, and Qwen are reserved in the adapter design but not connected.
- Prompt management is database-backed but has only the foundation seed prompt.
- Context builder is structured for future retrieval, but no semantic/vector retrieval is implemented yet.
- AI actions are internal/admin only at this stage.
- AI output remains advisory and does not apply changes to Product, Customer, Quote, PI, or Price Engine records.
