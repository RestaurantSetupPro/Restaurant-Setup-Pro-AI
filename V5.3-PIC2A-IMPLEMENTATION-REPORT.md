## 3. API Usage

PIC-2A reuses the existing Product Intelligence API:

```text
GET /api/product-intelligence/products
```

The Product Intelligence list API remains read-only and uses existing Product Library records.

The page also continues to load:

```text
GET /api/products
```

Purpose:

- Existing Product Create/Edit modal metadata
- Categories
- Tags
- SKU rules
- Existing Product Library form behavior

No new API endpoint was added.

### Existing API Field Extension

The existing Product Intelligence list response was extended with read-only fields needed by the Product List UI:

- `variant_count`
- `data_quality_status`
- `missing_fields`
- `ai_readiness_score`
- `ai_readiness_status`
- `pricing_status`
- `reference_price_display`
- `reference_price_min`
- `reference_price_max`
- `priced_variant_count`

These values are calculated from existing Product Library, Product Foundation, Product Readiness, and Product Price Engine data.

## 4. Permission Checks

PIC-2A preserves existing permission rules.

The Product Intelligence list API continues to use server-side redaction through existing sensitive-data protection.

Sales users can see:

- Product name
- Product image
- SKU
- Category
- Product status
- Variant count
- Sales-safe pricing status
- Data quality / AI readiness indicators

Sales users cannot receive supplier confidential information through the Product Intelligence API.

Admin / Owner users retain access according to existing permission rules.

No supplier cost column was added to the PIC-2A normal product table.
