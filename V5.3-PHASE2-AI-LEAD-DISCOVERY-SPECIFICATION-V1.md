## 1. Product Objectives

### 1.1 Business Goal

AI Lead Discovery Agent V1 should help companies discover valuable sales opportunities.

It should not be a simple customer list collector.

The system should help the user answer:

- Who should we sell to?
- Which companies are most likely to need our products?
- Why is this company a good opportunity?
- What should the salesperson do next?
- Which leads are not worth spending time on?

For Restaurant Setup Pro, the first target industry is hospitality furniture / restaurant furniture.

Typical target opportunities include:

- restaurant groups opening new stores
- hospitality design companies working on restaurant projects
- furniture distributors serving restaurants
- franchise operators
- bakery, coffee shop, bubble tea, bar, and fast casual operators
- hotel / food court / hospitality project buyers

### 1.2 Product Outcome

The agent should produce an opportunity pipeline that Sales can review.

Expected output:

- lead candidates
- AI opportunity cards
- opportunity score
- fit reasons
- buying signals
- missing data
- recommended next action
- human feedback status

The agent should improve over time using sales feedback, but AI must not become the source of truth.

### 1.3 Source-of-Truth Rule

The AI Lead Discovery Agent may create:

- discovery batches
- draft leads
- lead analysis
- lead score explanations
- ICP profiles
- recommendations
- logs
- feedback records

It must not silently overwrite:

- existing customers
- contacts
- opportunity scores
- quote or PI records
- product data
- price data

Any conversion from discovered lead to official customer/opportunity must require human review.

1. Company Information

## 3. Startup Modes

### 3.1 Mode A — Company Has Historical Customer Data

The user can upload historical customer records.

AI analyzes:

- customer type
- industry
- country
- city
- purchase pattern
- product category
- quote/order size
- repeat purchase behavior
- buying cycle
- common contact roles
- customer characteristics
- lost/won patterns

AI creates:

- Ideal Customer Profile
- high-value customer characteristics
- negative customer characteristics
- buying signals
- market patterns
- recommended discovery keywords
- recommended countries/cities

Historical data should be treated as learning context, not as perfect truth.

### 3.2 Mode B — Company Has No Historical Customer Data

The company can still start.

The user provides:

- company information
- product categories
- target markets
- customer types
- sales goals

AI creates an initial customer profile based on:

- product characteristics
- target industry rules
- known business logic
- market assumptions
- existing Product Intelligence Center data

The system should clearly label this as an initial profile.

Example status:

- `Initial ICP — based on company/product setup`

Once real customer feedback appears, the system should update the learning profile.

## 5. AI Customer Learning Profile

### 5.1 Purpose

The AI Customer Learning Profile stores what the system believes is a good customer for this company.

It should help future discovery, scoring, filtering, and sales recommendations.

The profile is not static. It should be improved through:

- historical customer analysis
- product fit analysis
- sales feedback
- won/lost patterns
- manual owner edits

### 5.2 Profile Components

The profile should include:

1. Ideal Customer Profile
2. High Value Customer Characteristics
3. Negative Customer Characteristics
4. Buying Signals
5. Product Fit Rules
6. Market Fit Rules
7. Contact Role Preferences
8. Avoid / Exclusion Rules

### 5.3 Ideal Customer Profile

Example:

```text
Industry: Hospitality Furniture

Ideal Customers:
- Restaurant Design Companies
- Hospitality Furniture Distributors
- Restaurant Groups
- Coffee Shop Chains
- Bubble Tea Chains
- Franchise Operators

Typical Need:
- commercial furniture package

- tables and chairs
- custom counters
- modular seating

Preferred Market:
- United States
- Canada
- Australia
- UAE

Preferred Project:
```

### 5.4 High Value Customer Characteristics

Examples:

- recent store opening announcement

## 6. AI Processing Logic

### 6.1 Processing Pipeline

Recommended V1 pipeline:

```text
Input Data
    ↓
Data Standardization
    ↓
Rule-Based Filtering
    ↓
ICP Matching
    ↓
Opportunity Scoring
    ↓
AI Explanation / Recommendation
    ↓
Human Review
    ↓
Approved Lead / Sales Queue
```

### 6.2 Data Standardization

Standardize:

- company name
- country
- city
- website
- phone
- email
- customer type
- industry
- source
- duplicate candidates

Standardization should run before AI calls.

### 6.3 Rule-Based Filtering Before AI

To control cost, obvious low-quality records should be filtered by rules before any AI analysis.

Rule examples:

- missing company name: Needs Review
- unrelated industry keyword: Low priority
- duplicate exact website: Duplicate
- no country and no website: Needs Review
- target country excluded: Excluded Market
- residential-only keywords: Low fit

Only leads that pass minimum quality should receive deeper AI analysis.

### 6.4 Staged AI Analysis

The system should not call AI for every field at once.

Recommended stages:

1. Fast Screening
   - rules only
   - basic fit
   - duplicate check
   - missing fields

2. Lightweight AI Fit Explanation
   - only for leads above minimum threshold
   - short prompt
   - low token limit

3. Deep AI Opportunity Analysis
   - only for selected high-potential leads
   - includes richer customer/product context
   - requires cost estimate and confirmation for batch runs

4. Sales Action Recommendation
   - generate next action
   - generate contact target suggestion
   - generate outreach angle

### 6.5 AI Business Brain Integration

Phase 2 should use the Phase 1 AI Business Brain foundation.

The Lead Discovery Agent should call the shared AI service contract rather than calling providers directly.

Example future action:

```text
runAiAction({
  moduleName: "lead-discovery",
  actionName: "lead.discovery.analyze",
  entityType: "lead_candidate",
  entityId: leadCandidateId,
  contextType: "lead_discovery",
  promptTemplateKey: "lead.discovery.analyze.v1",
  userId,
  options
})
```

The AI output should be stored as analysis/recommendation, not as final customer master data.

## 7. Opportunity Scoring Model

### 7.1 Score Range

Opportunity Score: 0–100

Recommended grades:

| Score | Grade | Meaning |
| --- | --- | --- |
| 90–100 | A+ | Highest priority, sales should review immediately |
| 75–89 | A | Strong opportunity, contact soon |
| 60–74 | B | Potential opportunity, needs more data |
| 40–59 | C | Low priority or uncertain fit |
| 0–39 | D | Not recommended for active sales time |

### 7.2 Scoring Factors

| Factor | Suggested Weight | Description |
| --- | ---: | --- |
| Customer Fit | 25 | Does the company match target customer types and ICP? |
| Product Fit | 20 | Is there likely demand for the company’s products? |
| Buying Signal | 20 | Is there evidence of opening, remodel, expansion, project work, or furniture need? |
| Company Size / Market Value | 15 | Store count, project scale, distributor/design firm value, market tier |
| Contact Availability | 10 | Email, website, phone, WhatsApp, decision-maker, contact role |
| Data Quality | 10 | Completeness, source confidence, duplicate confidence, freshness |

Total: 100

### 7.3 Customer Fit

High score examples:

- restaurant design company
- hospitality furniture distributor
- restaurant group
- coffee shop chain
- franchise operator
- bakery/cafe operator

Low score examples:

- residential-only buyer
- unrelated retail store
- individual consumer
- job seeker / service vendor

### 7.4 Product Fit

Product fit should be calculated using Product Intelligence Center categories and suitability rules.

Examples:

- Coffee Shop → dining chairs, small tables, booth seating, counter/service bar
- Restaurant → chairs, tables, booths, partitions
- Bar → bar stools, high tables, counters
- Outdoor dining → outdoor furniture
- Design Firm → full furniture package recommendation

The system must not copy Product Library data. It should store references and recommendation reasons.

### 7.5 Buying Signal

Signal examples:

- "opening soon"
- "new location"
- "remodel"
- "renovation"
- "expansion"
- "franchise"
- "project"
- "restaurant design"
- "hospitality interior"
- "furniture package"
- "DDP"

In V1, buying signals may come from imported notes, manually entered source text, website URL notes, or CRM export fields.

### 7.6 Contact Availability

Score higher if available:

- decision maker
- owner/founder
- purchasing manager
- operations manager
- design manager
- email
- WhatsApp / phone
- website contact page
- LinkedIn URL

### 7.7 Data Quality

Data quality should consider:

- required fields filled
- source confidence
- source freshness
- duplicate risk
- clear business type
- clear location
- traceable source URL or file

## 9. Human Feedback Loop

### 9.1 Purpose

The system should learn from sales feedback after follow-up.

Sales feedback should improve:

- ICP quality
- scoring accuracy
- negative customer detection
- recommended actions
- product fit rules
- future search keywords

### 9.2 Feedback Options

After salesperson follows up, available feedback should include:

- Interested
- Not interested
- Wrong customer
- Already purchased
- No response
- Future opportunity

Optional notes:

- reason
- contact quality
- budget signal
- timing
- product need
- competitor / current supplier

### 9.3 Feedback Effects

Feedback should update the learning profile as a recommendation, not silently overwrite rules.

Examples:

- "Wrong customer" reduces weight for similar customer type/keywords.
- "Interested" increases weight for similar signals.
- "Already purchased" may create nurture/future timing rule.
- "No response" should not immediately reduce customer quality unless repeated.
- "Future opportunity" should create a future follow-up recommendation.

### 9.4 Human Approval

Owner / Sales Admin should be able to review AI learning suggestions before applying major ICP rule changes.

This prevents one bad salesperson feedback item from distorting the whole system.

## 10. Token Cost Control Strategy

### 10.1 Core Cost Rule

All AI execution must pass through AI Cost Control and AI Business Brain.

No business module should call an AI provider directly.

### 10.2 Cost Reduction Strategy

Use this order:

1. Rule-based filtering
2. Duplicate detection
3. Cache check
4. Low-cost summary
5. AI analysis only for qualified records
6. Deep analysis only for selected high-potential leads

### 10.3 Staged Cost Levels

| Stage | Provider Need | Cost Strategy |
| --- | --- | --- |
| Import / Standardize | Rules | No paid AI required |
| Duplicate Check | Rules | No paid AI required |
| Basic Fit Score | Rules | No paid AI required |
| Fit Explanation | Mock/rules first, future LLM optional | Low token limit |
| Deep Opportunity Analysis | Future LLM optional | Confirmation required |
| Batch Discovery | Rules first, future LLM limited | Batch cost estimate required |

### 10.4 Caching

Cache rules:

- Same lead candidate + same source + same prompt version should not be re-analyzed unnecessarily.
- Same website URL should not be analyzed repeatedly within cache TTL.
- Same uploaded file row should not rerun deep analysis unless user chooses Regenerate.
- Feedback changes may invalidate part of the cache.

### 10.5 AI Call Limits

Recommended limits:

- Daily discovery AI budget
- Per-batch max AI analyzed leads
- Per-user daily run limit
- Max deep-analysis leads per run
- Max tokens per lead
- Admin/Owner override only

### 10.6 Execution Logs

Every AI-related lead discovery action should log:

- module name
- action name
- entity type
- entity id
- provider
- model
- prompt version
- context snapshot
- estimated cost
- actual cost
- cache status
- user
- status
- error

Logs should not expose secrets or provider API keys.

AI Lead Discovery Agent V1 should help Restaurant Setup Pro move from passive customer storage to proactive opportunity discovery.

The key product idea:

```text
Company/Product/Market Knowledge
        +
Optional Historical Customer Data
        ↓
AI Customer Learning Profile
        ↓
Lead Candidate Analysis
        ↓
Opportunity Cards
        ↓
Human Review + Sales Feedback
        ↓
```

- no external integrations yet
