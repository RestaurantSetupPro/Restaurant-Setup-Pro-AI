# AI Rules

## Scope

- Do not start a new module without explicit approval.
- Do not change accepted Module 01–04 behavior unless fixing an approved defect.
- Keep work focused on the requested module or operational issue.

## Data and Security

- Never expose or commit passwords, database URLs, tokens, API keys, or session values.
- Do not invent deployment, test, migration, or acceptance results.
- Report blocked external operations truthfully.

## Development

- Enforce the mandatory Migration, API, Debug, Test, and Documentation standard for every module.
- Inspect existing behavior before editing.
- Preserve unrelated user changes.
- Prefer normalized relationship tables over multi-value text fields.
- Keep SKUs unique and preserve role-based access controls.
- Add or update tests for behavioral changes.
- Run the relevant test suite before handoff.

## Delivery

- Lead with the verified outcome.
- List known issues and incomplete external steps.
- Stop after the requested delivery and wait for acceptance.
